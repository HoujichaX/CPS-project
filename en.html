First cup

Five minutes, one cup

Skip everything difficult for now. Take the simplest cup once around — pack, then bring it back. Your original file is never touched, so there's nothing to break.

Install CPS Preview — search "CPS Preview" on the Microsoft Store (free).
Drag a file in — open the app, go to the Compress tab, and drop any text file or image onto the window.✓ Your original stays where it is. Only a copy gets packed.
Press Pack — "Water" is already selected. Press Pack, pick a destination, and you have a .cps.
Bring it back — drop that .cps onto the Unpack tab and press Unpack. The exact same file comes back.
Try the next cup — same steps, but choose "Ice Cocoa" and the "all-round" dictionary. The closer the dictionary matches your data, the smaller the pack.

Basics

Pack it, open it

Two moves cover most days. They feel the same in every edition.

☕ Pack (compress)

  1. On the Compress tab, pick a file (drag & drop works)
  2. Choose a drink from the menu (Water / Ice Cocoa / …)
  3. Press Pack
  4. Choose where to save — done (.cps)

🫗 Open (unpack)

  1. On the Unpack tab, pick a .cps / .cpsv
  2. Press Unpack
  3. The method is auto-detected — your original file comes back
  4. Damaged parts are repaired with RS error correction when possible
Before using Hot Cocoa / Coffee: install a .cpsdict via Recipe Shelf → Add recipe. Layers are stored under %APPDATA%\cps\layers.

How-To

Every tab, gently

Following the sidebar, top to bottom. Each block starts with "what is this screen for?" so you always know where you are. Pro has every Standard tab plus three maker tabs (Workshop, Friend Key, Recipe Pack) — receiving (.cprp / friend keys) lives inside the Unpack tab on Pro.

Compress all editions

What is this screen for? — Packing a file into a cup to make a .cps. Your everyday screen.

  1. Pick a file — press Browse, or drop it anywhere on the window. Folders work too (they're bundled automatically).
  2. Pick a cup — from the method tabs. When in doubt, Water. Ice Cocoa shows a dictionary picker — choose whatever is closest to your content (or "all-round" if unsure).
  3. Hot Cocoa / Coffee / Shake show a recipe picker. If the list is empty, no recipe is installed yet — visit the Recipe Shelf first.
  4. For precious data, tick Rugged pack — a container that can heal itself (.cpsv).
  5. Press Pack, choose a destination, done. The log shows the shrunken size ("→ ○○ B").

If stuck: nothing happens → re-pick the file. "No recipe" → install one on the Recipe Shelf.

📷Screenshot: Compress tab with method tabs and a file selected

Unpack all editions

What is this screen for? — Turning a .cps / .cpsv back into the original file.

  1. Pick the file (drop works).
  2. Press Unpack. The cup it was packed with is detected automatically — you don't need to remember. Renamed files open fine too.
  3. Rugged packs attempt error-corrected repair if parts are missing or corrupted.
  4. On Pro this tab doubles as Receive — a self-contained pack (.cprp) opens by auto-detection like any file, and friend keys open via the dedicated fields at the bottom of the tab (point at both menu and .cpmn).

If stuck: "dictionary not found" → the recipe or pair generation used for this pack isn't installed yet. Ask the sender, or import it via Receive / Pairs.

📷Screenshot: Unpack tab with a completed log

Recipe Shelf Standard / Pro

What is this screen for? — The shelf where recipes (.cpsdict) for Hot Cocoa / Coffee / Shake live.

  1. Press Add recipe and pick a .cpsdict — it lands on the shelf (stored in %APPDATA%\cps\layers).
  2. Installed recipes appear in the pickers on the Compress tab.
  3. Start with the bundled sample_layers\ (described under Dictionaries below) — Standard and Pro install them to the shelf automatically on first launch.
  4. On Pro the shelf works with the Workshop — add cocoa/coffee .cpsdict files you baked; Shake layers are installed automatically at bake time, so no extra step. "Create memo" saves a Shake layer set with its order.
📷Screenshot: Recipe Shelf with a few entries

Receive Standard

What is this screen for? — The counter where special packs from other people are opened.

  1. Got a self-contained pack (.cprp)? Just open it — the recipe travels inside, nothing to prepare.
  2. Got a Friend Key? It arrives as a set of two files: the .cpmn (contents) and the menu (key info). Point at both to unpack — one alone won't open.
  3. Pro has no Receive tab — do the same from the Unpack tab (.cprp opens by auto-detection; friend keys via the fields at the bottom).
📷Screenshot: Receive tab

Pairs all editions (tiered)

What is this screen for? — Growing a dictionary that belongs to just you and one partner, then sending small, private packs. The most interesting part of CPS.

  1. On Pro (the maker): register a partner with + New pair → pick sample files close to your real exchanges and press Grow pair dictionary → when grown, Share dictionary exports a .cpgen to send them.
  2. On Standard (receive & send): import the received .cpgen with "Receive dictionary" — then "Pack & send" to that pair becomes available. Three ways to send:
    • Auto (efficient) — packs the smallest, ordinary way (.cpp2). When unsure, pick this.
    • Secret (private) — only someone holding the pair dictionary can open it (.cpp3). Not encryption.
    • Keyed (per-message key) — Secret plus one extra key for that single pack (.cpp3l); hand the key file over separately.
  3. Pro has one more: "Hoard (self-archive)" — a preset for growing, not sending. It keeps a larger learning pool so you can fatten a self-to-self backup pair over time (growing is Pro-only, so this option only appears on Pro).
  4. When to grow, when to stop:
    • What "growing" actually costs — sending the new generation dictionary (.cpgen, ~256 KB by default) to your partner again. The packs themselves never get heavier as generations advance.
    • Grow when — the pair is young (the first few generations gain the most), or the topic of your exchanges shifts (new project, new vocabulary). Packs shrinking less than they used to is the "dictionary is stale" signal.
    • Stop when — new generations stop improving the ratio: the dictionary is saturated (our mail tests plateaued around 256 KB). Just stop growing — no "off switch" needed. The latest generation your partner holds keeps working forever, at zero further transfer cost.
    • Rule of thumb — grow only if the total savings on future packs will exceed the size of the generation you'd send. No need to grow per message; batching dozens of exchanges into one new generation is plenty. On Pro, pick a folder to bundle everything inside into a single new generation at once.
    • Distributing to many people? — pair dictionaries are strictly two-person. For wider distribution use a Recipe Pack (.cprp) or a plain .cps instead.
  5. On Preview: import generations and open pair packs (Keyed too, with the key file). No compressing or sending with the pair dictionary.
  6. Opening picks the right dictionary automatically from the lineage and generation stamped on the pack. Growing never breaks old packs — old generations stay in the vault.

If stuck: Keyed packs won't open without the key file. Hand the key over separately from the pack.

Workshop Pro

What is this screen for? — Baking recipes (.cpsdict) from your own data. The screen reads top to bottom: material → kind → name → size → detail → bake.

  1. Pick the material — "File…" for one file, or "Folder…" for a whole folder. A folder is learned as a bundle (use this for mail archives and log piles — a structure that appears once per file becomes a learnable pattern when bundled).
  2. Pick the kind — Hot Cocoa / Coffee / Shake on the tabs at the top. See "Recipe Tips" for which fits your data.
  3. Name the recipe — shown on the shelf and in memos.
  4. Size — Light (256 KB) / Standard (1 MB) / Full (2 MB) / Custom. Bigger means more vocabulary, but gains plateau. Standard is a fine default.
  5. Detail (optional) — Min/Max = phrase length range, Freq = occurrences required (defaults 4 / 64 / 3). With Shake the meaning changes: Min = boundary of the long layer, Max = boundary of the short layer (defaults switch to 16 / 4 automatically).
  6. LLM curation (optional) — tick to have an LLM filter candidate phrases. Choose a provider and enter your API key (your key; your material is sent). "Save key" remembers it. Recipes bake fine without it.
  7. Press "Make recipe" — Cocoa/Coffee: choose where to save and you get a .cpsdict; add it on the Recipe Shelf to use it yourself, or hand the file to a Standard user. Shake: long/mid/short layers are baked and installed to the shelf automatically, together with a recipe memo that remembers their order (no save dialog).

If stuck: "no frequent phrases found" → your material is too small. Add more same-genre data or lower Freq.

Glossary

The hard words, softly

Q&A for the vocabulary that shows up in CPS.

What is a dictionary?

A lookup table that swaps frequent words and patterns for short codes. The closer the dictionary matches your content, the smaller it packs — same trick as human shorthand.

Recipes? Layers? How do they differ from dictionaries?

All members of the same family. Hot Cocoa knows word substitutions, Coffee knows line shapes (for log-like data where the same shape repeats), Shake knows long runs. They can be stacked, hence "layers". On disk, they're all .cpsdict files.

What is a pair "generation"?

A version number that goes up each time the pair dictionary is grown. Newer generations know your exchanges better. Old packs open with old generations, so growing never locks you out of past packs.

Is "Secret" encryption?

No. It's limited disclosure — hiding content and patterns from outside eyes, with the pair dictionary acting as the shared key. For a message you truly want locked, use Keyed and pass the key file separately.

What's the point of the Rugged pack?

A container that resists breakage: if parts get lost or corrupted in storage or transfer, error correction can often restore them. For long-term storage and precious hand-offs.

Does compressing delete or degrade my file?

Neither. Your original stays untouched; a .cps simply appears next to it. Unpacking returns a byte-identical file (lossless).

Three ways to send

Efficient .cpp2

The everyday cup — smallest size with your pair dictionary.

Secret .cpp3

Substitution + 4D shuffle hides content and patterns. No key exchange needed — the pair dictionary is the first key.

Keyed .cpp3l

Secret plus a per-message key on top. Only someone with both the dictionary and the key can open it. Hand the key over separately.

Secret is not encryption. It is limited disclosure — hiding plaintext and patterns. For a message you truly want locked down, use Keyed (.cpp3l) and pass the key another way. Creating pairs and growing dictionaries is Pro; sending with a received dictionary is Standard; importing and opening works even in the free Preview.

Recipe making

Baking recipes that actually work

Measured guidance for the Pro Workshop — how to bake recipes that really shrink things. Most important first.

The golden rule — the ingredients are everything

Recipe quality is decided by what you feed it, not the algorithm. Use content close to what you'll compress, in bulk. In our tests, a cocoa baked from 155 KB of same-genre documents beat plain Water even on unseen files; one baked from a 61 KB grab-bag had zero effect. Small and messy fails; homogeneous and plentiful wins. More is better — think hundreds of KB and up.

Pick the method to match the data

Ordinary documents and articles → Hot Cocoa (word substitution). Logs where the same line-shape repeats with changing numbers → Coffee (line templates). Long boilerplate blocks (signatures, headers) recurring far apart → Shake (chunk substitution). And know when to abstain: chat-style data of short unique messages resists substitution recipes — plain Water or Ice Cocoa compresses it better.

Cocoa — the forgiving cup

Cocoa doesn't substitute; it melts the dictionary into the compressor as context. A mismatched dictionary costs almost nothing; a matched one wins big. Redundant phrases in the dictionary do no harm, so don't fuss — just gather same-genre material. Bigger isn't endlessly better either: gains plateau (the default 256 KB–1 MB is usually plenty).

Coffee — bundle files so one-off structures become patterns

A structure like an email header appears once per message, so a single file can't teach it. But bundle dozens of same-format files and it appears dozens of times — now it's a learnable template. Coffee's ingredient is "a stack of files in the same format": logs, ledgers, uniform records. Not single files, not prose.

Shake & personal recipes — big dictionary, small data

The winning pattern: hand over a large dictionary once, then send small data many times. A long chunk stored in the dictionary shrinks to one reference no matter how far apart it recurs. In our tests, a personal recipe baked from a mail archive improved compression by 26% (signatures and boilerplate did the work). If the data itself is large, the compressor finds the repeats on its own and the recipe loses its job — which is exactly why pair dictionaries shine for two people exchanging many small messages.

Advanced settings — the only knobs worth touching

Three knobs in the Workshop. Min/max length bound the phrases collected (default 4–64). Frequency is "how many occurrences before adoption" (default 3; lower it and vocabulary grows, but coincidences creep in). For mail-like data with long far-apart boilerplate, dropping frequency to 2 and allowing longer phrases is the classic move. When unsure, defaults are fine.

Don't worry about time — baking is slow, opening is always fast

Take your time baking and packing (they're one-time jobs). Unpacking speed is unaffected by dictionary size — always fast. So the right way to use CPS: bake a good recipe slowly, pack carefully, and the receiver opens it in an instant. A slow bake is quality, not a problem.

File types

Extension cheat sheet

  • .cpsStandard pack — one of the five methods, auto-detected on open.
  • .cpsvRugged pack. A frame + RS container built to survive damage.
  • .cpsdictLayer / dictionary. Install from the Recipe Shelf.
  • .cpgenPair generation pack — hands a grown dictionary to your partner (lineage + generation inside).
  • .cpp2Pair pack (Efficient). Smallest everyday pack with the pair dictionary.
  • .cpp3Pair pack (Secret). Substitution + 4D shuffle, limited disclosure.
  • .cpp3lPair pack (Keyed). Secret plus a per-message key — two locks.
  • .cprpSelf-contained pack with the recipe inside. Opens with nothing installed.
