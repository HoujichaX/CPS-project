CPS はファイルを「飲みもの」に見立てて圧縮する Windows アプリです。 辞書のいらない一杯から、こだわりの一杯まで。壊れにくい瓶づめや、フレンドとの受け渡しもできます。

はじめて

まずは5分、ためしに一杯

むずかしいことは全部あとまわし。いちばん簡単な一杯で「詰めて、戻す」を一周してみましょう。元のファイルは消えないので、安心して試せます。

CPS Preview を入れる — Microsoft Store で「CPS Preview」を検索してインストール(無料)。
ファイルをドラッグ — アプリを開いて「圧縮」タブに、適当なテキストファイルや画像を1つドラッグ&ドロップ。✓ 元のファイルはそのまま残ります。コピーが詰められるだけ。
そのまま「梱包する」 — 方式は最初から「ミネラルウォーター」が選ばれています。押して、保存先を決めるだけ。.cps ができたら成功。
戻してみる — 「開封」タブにさっきの .cps をドラッグして「開封する」。元とまったく同じファイルが戻ってきます。
次の一杯へ — 同じ手順で、今度は方式を「アイスココア」にして辞書「全部入り」を選んでみましょう。中身に合う辞書ほど小さくなるのが分かります。

メニュー

使い方

詰める・開ける、基本の2つ

まずはこれだけ。どのグレードでも同じ手触りで動きます。

☕ 詰める(梱包)

  1. 「梱包する」タブでファイルを選ぶ(ドロップでも可)
  2. メニューから一杯を選ぶ(水 / アイスココア / …)
  3. 「梱包する」を押す
  4. 保存先を決めて完成(.cps)

🫗 開ける(開封)

  1. 「開封する」タブで .cps / .cpsv を選ぶ
  2. 「開封する」を押す
  3. 方式は自動判定 — 元のファイルに戻る
  4. 途中が欠けていてもRS 誤り訂正で復元を試みます
ココア / コーヒーを使う前に: 「レシピ棚 → レシピを追加」で .cpsdict をインストールしてください。 Standard には同梱の sample_layers\ (お試し用11個)が入っています。インストール先は %APPDATA%\cps\layers です。

瓶づめは梱包タブで「瓶づめ」を選ぶだけ。外殻はフレーム単位で少しずつ書き込むので、巨大なファイルでも安定します。

Preview 版では内部の CPS を含めて 1.5 GiB までが対象です。開封は中身を見て自動判定するため、拡張子を変えても開けます。

ペアリング

二人で、育てていく辞書

特定の相手と「ペア」を組むと、やり取りするほど二人だけの辞書が育ちます。世代を重ねるごとに賢くなり、 同じ相手へのパックはどんどん小さくなっていく——それがペアリングです。

送り方は3つ

効率 .cpp2

ペア辞書でいちばん小さく詰める、日常づかいの一杯。

ひみつ .cpp3

置換＋4Dシャッフルで中身とパターンを伏せる。鍵の受け渡しは不要(ペア辞書が第一の鍵)。

かぎつき .cpp3l

ひみつの上に「1通ごとの鍵(フレンドkey)」を重ねた二層。辞書と鍵の両方を持つ人だけが開けます。鍵は別便で手渡し。

「ひみつ」は暗号ではありません。 平文とパターンを伏せる限定開示です。強く限定したい1通は「かぎつき(.cpp3l)」にして、鍵を別に渡してください。 ペアを作る・辞書を育てるのは Pro、受け取った辞書で送るのは Standard、受け取って開くだけなら無料 Preview でできます。

操作ガイド

タブごとの、ていねいな手順

サイドバーのタブに沿って、上から順に。各タブの「これは何をする画面？」から読めば迷いません。Pro は Standard の全タブ＋作り手用3タブ(工房・鍵かけ・レシピパック)の構成です。受け取り(.cprp/フレンドkey)は Pro では「開封」タブに統合されています。

圧縮 全グレード

これは何をする画面？ — ファイルを一杯に詰めて .cps を作る画面。いちばんよく使います。

  1. ファイルを選ぶ — 「参照」を押すか、ウィンドウのどこでもいいのでドラッグ&ドロップ。フォルダごとでもOK(自動でひとまとめになります)。
  2. 一杯を選ぶ — 上部の方式タブから。迷ったら「ミネラルウォーター」。アイスココアにすると辞書の選択が出るので、中身に近いものを(分からなければ「全部入り」)。
  3. ココア/コーヒー/シェイクは、インストール済みのレシピを選ぶ欄が出ます。一覧が空ならまだレシピが入っていない合図 — 先に「レシピ棚」へ。
  4. 大切なデータなら「瓶づめ」にチェック。壊れても直せる頑丈な容器(.cpsv)になります。
  5. 「梱包する」を押して保存先を決めたら完成。ログに「→ ○○ B」と縮んだサイズが出ます。

つまずいたら: 何も起きない→ファイルを選び直す。「レシピがありません」→レシピ棚でインストール。

📷スクリーンショット: 圧縮タブ(方式タブとファイル選択が見える状態)

開封 全グレード

これは何をする画面？ — .cps / .cpsv を元のファイルに戻す画面。

  1. ファイルを選ぶ(ドラッグでもOK)。
  2. 「開封する」を押すだけ。どの一杯で詰めたかは自動で判定されるので、覚えていなくて大丈夫。拡張子を変えられていても開きます。
  3. 瓶づめ(.cpsv)は、途中が欠けたり化けたりしていても誤り訂正で復元を試みます。
  4. Pro ではこのタブが「受け取り」も兼ねます — 自己完結パック(.cprp)はそのまま選べば自動判定で開き、フレンドkey はタブ下部の専用欄(menu＋.cpmn の2ファイル指定)で開封します。

つまずいたら: 「辞書が見つかりません」→そのパックに使われたレシピや世代辞書がまだ入っていません。作った人に聞くか、受け取り/ペアで取り込みを。

📷スクリーンショット: 開封タブ(完了ログが出た状態)

レシピ棚 Standard / Pro

これは何をする画面？ — ココア/コーヒー/シェイク用の「レシピ」(辞書ファイル)を入れておく棚。

  1. 「レシピを追加」を押して .cpsdict ファイルを選ぶと、棚に入ります(保存先: %APPDATA%\cps\layers)。
  2. 入れたレシピは、圧縮タブのココア/コーヒー/シェイクの選択欄に現れます。
  3. 最初は同梱の sample_layers\ から入れてみましょう(中身は下の「同梱辞書」で説明)。Standard / Pro とも初回起動時に自動で棚に並びます。
  4. Pro では工房と連携します — 工房で焼いたココア/コーヒーの .cpsdict をここで追加。シェイクは焼いた時点で自動導入されるので追加操作は不要です。「メモを作る」でシェイクの層構成(適用順)を保存できます。
📷スクリーンショット: レシピ棚(一覧に数件入った状態)

受け取り Standard

これは何をする画面？ — 人からもらった特別なパックを開ける窓口。

  1. 自己完結パック(.cprp)をもらったら — 選んで開くだけ。レシピが同梱されているので、何も準備が要りません。
  2. フレンドkey をもらったら — .cpmn(中身)と menu(鍵情報)の2つセットで届きます。両方を指定して開封。片方だけでは開きません。
  3. Pro にはこのタブはありません — 同じ操作を「開封」タブで行います(.cprp は自動判定、フレンドkey は開封タブ下部の欄で)。
📷スクリーンショット: 受け取りタブ

ペア 全グレード(できることは段階制)

これは何をする画面？ — 特定の相手と「二人だけの辞書」を育てて、小さく・ひそかに送り合う画面。CPSのいちばん面白いところ。

  1. Pro の人(作る側): 左下の「＋ 新規ペア」で相手の名前を登録 → 二人のやり取りに近い素材ファイルを選んで「このペアの辞書を育てる」→ 育ったら「辞書を渡す」で .cpgen を書き出して相手に送る。
  2. Standard の人(受け取って送る側): 「辞書を受け取る」でもらった .cpgen を取り込む → 以降そのペアあてに「梱包して送る」が使えます。送り方は3つ:
    • おまかせ（効率） — いちばん縮む形でふつうに梱包(.cpp2)。迷ったらこれ。
    • ひみつ（限定開示） — ペア辞書を持つ相手だけが開ける形(.cpp3)。※暗号ではありません。
    • かぎつき（1通ごとの鍵） — ひみつの上に、その1通専用の鍵をもう一枚(.cpp3l)。鍵ファイルは別の経路で手渡し。
  3. Pro にはもう1つ「ためこみ（自分用保管）」があります — 送る用ではなく育てる用のプリセット。学習素材のプールを厚めに持ち、自分⇔自分のバックアップペアをじっくり太らせたいときに選びます（辞書を育てられるのは Pro だけなので、この選択肢も Pro だけにあります）。
  4. 育てどきの目安（いつまで育てる？いつ止める？）:
    • 「育てる」の通信コストの正体 — 新しい世代辞書(.cpgen・既定で256KB前後)を相手に送り直すこと、それ自体です。パックの送信は世代を重ねても重くなりません。
    • 育てるべきとき — 付き合い始め(最初の数世代は伸びが大きい)と、やり取りの話題が変わったとき(新しい仕事・新しい用語)。パックの縮みが悪くなってきたら「辞書が古い」サインです。
    • 止めどき — 世代を重ねても縮み方が変わらなくなったら飽和です(メール系は256KBで頭打ちという実測があります)。そこで育てるのをやめれば、以降の辞書転送コストはゼロ。「オフにする」操作は不要で、相手が持つ最新世代はそのままずっと効き続けます。損はありません。
    • 目安の式 — 「新世代を送る量」より「今後のパックで節約できる量の合計」が大きいなら育てる価値あり。毎回育てる必要はなく、数十通ためてからまとめて一世代、で十分です。Pro の「育てる」はフォルダを選べば中のファイルを束ねて一気に1世代にできます。
    • 大勢に配りたいとき — ペア辞書は二人専用です。不特定の相手に配るならペアではなくレシピパック(.cprp)か、ふつうの .cps を使ってください。
  5. Preview の人: 世代辞書の取り込みと、ペアパックの開封まで（かぎつきは鍵ファイルがあれば開封可）。ペア辞書での圧縮・送信はできません。
  6. 開くときは、パックに刻まれた系譜と世代から自動で正しい辞書が選ばれます。辞書を育てても、昔のパックはちゃんと開きます(古い世代が保管されているため)。

つまずいたら: 「かぎつき」は鍵ファイルがないと開きません。鍵はパックとは別の方法で受け渡しを。

📷スクリーンショット: ペアタブ(台帳に1件ある状態)

レシピ工房 Pro

これは何をする画面？ — 自分のデータからレシピ(.cpsdict)を焼く工房。画面は上から「素材 → 種類 → 名前 → サイズ → 詳細 → 焼く」の一本道です。

  1. 素材を選ぶ — 「ファイル…」で1つ、または「フォルダ…」で丸ごと。フォルダを選ぶと中のファイルを束ねて学習します(メールの束・ログの束はこちら。1ファイルでは1回しか出ない構造も、束ねれば「型」として学べます)。
  2. 種類を選ぶ — 上部のタブで ホットココア / コーヒー / シェイク。素材との相性は「レシピ作りのコツ」を参照。
  3. レシピ名を付ける — 棚やメモに表示される名前です。
  4. サイズ — 手軽(256KB) / 標準(1MB) / 本格(2MB) / カスタム。大きいほど語彙が増えますが、ある点から先は頭打ちです。迷ったら標準。
  5. 詳細(任意) — 最短/最長=拾うフレーズの長さ範囲、頻度=採用に必要な出現回数(既定 4 / 64 / 3)。シェイク選択時だけ意味が変わり、最短=長い層との境界・最長=短い層との境界になります(既定 16 / 4 に自動で切り替わります)。
  6. LLMキュレーション(任意) — チェックすると候補フレーズをLLMが選別します。提供元を選んでAPIキーを入力(自分のキー・素材が送信されます)。「キーを保存」で次回から入力不要。使わなくてもレシピは焼けます。
  7. 「レシピを作る」を押す — ココア/コーヒー: 保存先を選んで .cpsdict が完成。レシピ棚で追加すれば自分でも使え、ファイルを渡せば Standard の人も使えます。シェイク: 長/中/短の3層が自動で焼かれて棚に導入され、順序を覚えたレシピメモも一緒に作られます(保存先選択は不要)。

つまずいたら: 「頻出フレーズが見つかりません」→素材が少なすぎます。同種のデータを足すか、頻度を下げてみてください。

📷スクリーンショット: レシピ工房(焼き上がりログ)

鍵かけ・レシピパック Pro

これは何をする画面？ — 焼いたレシピや荷物を「配る」ための2タブ。

  1. 鍵かけ: 相手だけが開ける限定公開パック(.cpmn+menu)を作ります。2ファイルセットで渡してください。
  2. レシピパック: レシピを同梱した .cprp を作ります。受け取る側は何も入れずに開けるので、配布にいちばん親切。

ことば

むずかしい言葉を、やさしく

CPSに出てくる言葉の意味を、Q&Aで。

辞書って何？

「よく出てくる言葉や並び」を短い符号に置き換えるための対応表です。中身に合った辞書を使うほど、同じデータがぐっと小さくなります。人間の速記と同じ理屈。

レシピとかレイヤーって何？辞書と違うの？

どれも辞書の仲間です。ココアは「言葉の置き換え」、コーヒーは「行の形」(ログのように同じ型の行が並ぶデータ向け)、シェイクは「長い塊」を覚えたレシピ。重ねて効かせられるので「レイヤー」とも呼びます。ファイルとしてはぜんぶ .cpsdict です。

ペアの「世代」って何？

二人の辞書を育てるたびに増える「版」の番号です。世代が進むほど、二人のやり取りに詳しい辞書になります。古いパックは古い世代の辞書で開かれるので、育てたせいで昔の荷物が開かなくなることはありません。

「ひみつ」は暗号なの？

暗号ではありません。中身と並びのパターンを外から見えにくくする「限定開示」です。ペアの辞書そのものが合鍵の役割をします。本気で守りたい1通は「かぎつき」にして、鍵ファイルを別の方法で渡してください。

瓶づめって何がうれしいの？

保存中や受け渡しでファイルの一部が欠けたり化けたりしても、誤り訂正の力で元に戻せる可能性が高い、割れにくい容器です。長期保存や大事なデータの受け渡しに。

圧縮すると元のファイルは消える？中身は劣化する？

どちらもありません。元のファイルはそのまま残り、.cps がもう1つできるだけです。開封すれば1バイトも違わない元のファイルが戻ります(可逆圧縮)。

レシピ作り

効くレシピを焼くコツ

Pro のレシピ工房で「本当に縮む」レシピを作るための、実測にもとづく指針です。いちばん大事なことから順に。

大原則 — 素材がすべて

レシピの良し悪しは、アルゴリズムではなく食わせる素材で決まります。「圧縮したいものに近い内容」を「まとまった量」。実測では、同じ種類のドキュメント155KBで焼いたココアは初見のファイルでも水より縮みましたが、バラバラな寄せ集め61KBで焼いたものは効果ゼロでした。少量・雑多はダメ、同種・大量が正解。多いほど良く、目安は数百KB〜です。

方式えらび — データの「効きどころ」に合わせる

同じデータでも効く方式は違います。ふつうの文書・記事・ドキュメント → ココア(言葉の置き換え)。同じ形の行が数値だけ変えて並ぶログ → コーヒー(行の型)。長い定型の塊(署名・ヘッダ・ボイラープレート)が飛び飛びに繰り返す → シェイク(塊の置換)。逆に、チャットや雑談ログのような「短い発言の集まり」は置換レシピと相性が悪く、素の水やアイスココアの方が縮みます。無理にレシピを当てないのも腕のうち。

ココアのコツ — 外しても害がない、気楽な一杯

ココアは辞書を「置き換え」に使わず、圧縮機に文脈として溶かし込む方式。だから辞書がデータと合わなくても悪化はほぼせず、合えば大きく縮みます。冗長なフレーズが辞書に混ざっていても害はありません。細かいことは気にせず、同種の素材を集めることに集中してください。サイズは大きくすれば良いわけではなく、ある点から先は頭打ちになります(既定の256KB〜1MBで十分なことが多い)。

コーヒーのコツ — 1回しか出ない構造は「束ねて」型にする

メールの先頭のような構造は、1通の中では1回しか出ないので学習できません。でも同じ形式のファイルを何十個も束ねて工房に渡せば、全体では何十回も現れて「型」として学習できます。コーヒーの素材は「同じフォーマットのファイルの束」——ログ、帳票、同一形式のレコード。1個のファイルや散文には向きません。

シェイク・個人レシピのコツ — 辞書は大きく、データは小さく

シェイク系の勝ちパターンは「大きい辞書を一度渡して、小さいデータを何度も送る」。辞書にある長い塊が小さなデータに現れたら、距離に関係なく一発で縮みます。実測ではメールの束から焼いた個人レシピで26%の改善が出ました(署名・定型の塊が効いた)。逆にデータ側が大きいと圧縮機が自力で反復を見つけてしまい、レシピの出番が消えます。ペア辞書が「小さなやり取りを重ねる二人」に効くのは、この性質ゆえです。

詳細設定の意味 — 触るならここだけ

工房の詳細設定は3つ。最短/最長は拾うフレーズの長さの範囲(既定 4〜64)。頻度は「何回以上出たら採用するか」(既定 3。下げると語彙が増えるが、たまたまの一致も拾いやすくなる)。メールのような「遠くで繰り返す長い定型」を狙うときは、頻度を2に下げて長めを許すのが定石です。迷ったら既定のままで大丈夫。

時間の心配は不要 — 焼きは遅くてOK、開封はいつも速い

レシピ作りと圧縮には時間をかけて構いません(一度きりの作業)。開封の速さは辞書のサイズに影響されず、常に高速です。だから「時間をかけて良いレシピを焼き、じっくり詰めて、受け取った人は一瞬で開ける」が CPS の正しい使い方。焼き上がりが遅くても、それは品質のための時間です。

Preview の上限: 梱包・開封とも、内部データを含めて 1.5 GiB までです。それより大きなデータは Standard / Pro の公開をお待ちください。
Preview（無料）でできないこと: 無料版は「気軽に試す」ためのもので、次の機能は含まれていません。必要な場合は Standard / Pro の公開をお待ちください。
  • 圧縮はミネラルウォーターとアイスココアの2つだけ。 ホットココア・コーヒー・シェイクでは詰められません。
  • レシピ棚がありません。 同梱のお試し辞書や自作の .cpsdict をインストールして使うことはできません（＝辞書を使う圧縮ができません）。
  • ペアは「受け取って開く」だけ。 届いた世代辞書を取り込んで、ペアパックを開けます — かぎつき（.cpp3l）も鍵ファイルがあれば開けます。ただしペア辞書を使った圧縮・送信（独自辞書での圧縮）はできません。ペアを新しく作る・辞書を育てるのは Pro、ペアあてに送る（.cpp2/.cpp3/.cpp3l の作成）は Standard 以上です。
  • レシピ工房・鍵かけ・レシピパックはありません。 辞書やレシピ、限定公開パックを作る機能は Pro 専用です。

開封は制限なしで、あらゆる方式の .cps / .cpsv を開けます（他の人が作ったものも）。「作る側」の機能が上位グレード、と考えるのが近いです。
